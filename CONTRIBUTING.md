# Contributing

Thanks for considering contributing to this project. A few quick guidelines to keep contributions consistent:

- Fork the repository and create a feature branch from `main`.
- Keep commits small and focused. Use descriptive commit messages.
- Open a pull request and include a short description of your changes and why they are needed.
- Run linting before submitting: from `frontend` run `npm run lint` and `npm run format`.

Code style: follow the existing ESLint rules in `frontend` and keep changes minimal to the code's style.

If your change requires environment variables, document them in `README.md` or add an example `.env.example` file in the relevant folder.
